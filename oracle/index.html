Index (i/x)

* is an optional data structure that u create on one or more columns of a table.
* increases the performance of data retreival.
* when processing a request/running a query d/b can use available indexes to locate requested rows efficiently.
* logically & phsically independent of data.
    * u can drop & create with on effect on tables or other indexes.
++++++++++++++++++++++++
* by creating an index on one or more columns of a table, u gain ability in some cases to retrieve small set of randomly distributed rows of a table.
* i/x r one of many means of reducing disk I/O.


Key & Columns
* key is set of columns or expressions on which u can build an i/x.
* i/x r structures stored in d/b tht users manage using SQL. Key r strictly a logical concept.
* primary & unique keys automatically have i/xs
    * u might want to create i/x on foreign key.
