collection
* is an ordered group of elements (of same type)
* is a general concept that encompasses lists, array, and other datatypes used in classic programming algorithms
* each element is addressed by unique subscript
* collection types
  1. associative arrays
  2. nested tables
  3. varrays


record
* group of related items stored in fields, each with its own name & datatype.
* its like a variable that can hold a table row (or some columns of table row)
* %rowtype attribute : to declare a pl/sql record that represent a row in d/b table (for all columns of the row)
* for some columns or columns from different tables, define a view or declare a cursor to select right columns & necessary joins & then apply %rowtype to view or cursor.

defining & declaring records
* to create records, define a RECORD type & then declare records of that type.
    or
* create or find table,view or cursor & use %ROWTYPE attribute
        * like var1        employees%rowtype       for employee table record.
                  var2        employee_v%rowtype     for view
                  var3        c1%rowtype                     for cursor
