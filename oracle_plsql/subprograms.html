Subprograms

* subprograms r the named pl/sql blocks that can be called with a set of parameters.
* types of subprograms:
  1. procedures
  2. functions
* similar to anonymous blocks, subprograms have:
  1. declarative part
  2. executable part
  3. exceptional-handling part (optional)

why to use subprograms (advantages)


procedure (p/c)
* p/c is a subprogram that performs a specific action.
* specify procedure name, its parameter, its local variables, BEGIN-END block & exception part
* can specify whether p/c executes using schema & permissions of the user who defined it or user who calls it (check AUTHID clause for this).
* can specify whether p/c should be part of current transaction, or execute in its own transaction
        where it can COMMIT or ROLLBACK withoud ending transaction of caller

functions
* f/n is a subprogram that computes a value.
* f/n & p/c r strutured alike, except that f/n have a RETURN clause.
* PARALLEL_ENABLE option
* DETERMINISTIC option
* AUTONOMOUS_TRANSACTION pragma

parameter (p/m)
* for each p/c u specify:
  1. p/m name
  2. p/m mode
  3. p/m datatype
  4. p/m default value (optional)
* formal parameter
* actual parameter
* when calling a subprogram u can write actual p/m using:
  1. positional notation
  2. named notation
  3. mixed notation
* p/m modes: use to define the behaviour of formal p/ms
        * three p/m modes r:
                1. IN
                2. OUT
                3. INOUT
RETURN statement
* do not confuse RETURN statement with RETURN clause in f/n specification.
* RETURN statement immediately ends execution of a subprogram and return control to caller.
