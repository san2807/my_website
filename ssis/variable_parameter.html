Variables (v/a)
* contains a value that can b hardcoded, dynamically set once or modified multiple times throughout execution of package.
* provides a method for objects in a package to communicate each other.
* similar to their use in programming languages, variables r used in specific types of logic
    (such as iterating through a loop, concatenating multiple values together to create a file d/y
     or passing an array bw objects)
* v/a can b created, deleted, renamed and have their data types changed, as long as p/c in design mode.
* once p/c is validated & in runtime mode, v/a definition in locked ; only value of v/a can be changed.
* scope of v/a : v/a can be scoped either to a p/c
                                                        or to a specific component in the p/c
* v/a r case sensitive
* v/a can hide other v/a values higher in the hierarchy.
