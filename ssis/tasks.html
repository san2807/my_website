Control Flow

Control flow SSIS toolbox contains 3 types of objects:
  1. Containers
  2. Control flow items/tasks
  3. Maintenance Plan tasks


Containers
* special type of item that can hold other control flow items/tasks & maintenance plan tasks [sanb1]
* help us to repeat a set of tasks or organize group of tasks within in p/c [sanb1]

* core units in SSIS architecture from grouping tasks together logically into units of work.
* besides visual consistency, it enable u to define variables & event handlers within the scope of container (instead of package)

  1. Task Host
  2. Sequence
  3. For Loop
  4. Foreach Loop


Control Flow Tasks

* task can best described as an individual unit of work.
* task is discrete unit of work that can perform typical actions required by an ETL process, from moving a file and preparing data sources
    to sending e-mail confirmation when everything is complete.
* more advanced tasks enable u to perform actions like executing SQL commands, sending mails, running VB or C# code

* task provide functionality to ur package (in the same way method does in programming language)
* task r extensible. u can create ur own custom task in .NET
* create ur own task if u need a workflow item that doesn't exist or if u have a common scripting function that can benefit from reuse
    in ur package development

For Loop Container
* enables us to repeat a segment of a control flow. [sanb1]

Foreach Loop Container
* like For loop, it also provides a way of repeating a segment of control flow
        - but rather than have an expression to control when the loop is exited, Foreach Loop container iterates
        one time for each item in collection. [sanb1]

Bulk Insert

Data Profiling
* gathers information about the set of data [sanb1]
* can be used to determine information about the particular set of data we intend to process through an int services p/c [sanb1]
* profiling operations u can do with this task. [sanb1]
* data profiling task outputs its findings to an XML file. [sanb1]
    * this xml file can b viewed using data profile viewer. [sanb1]

+++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++

* data profiling is the process of examining data and collecting metadata about quality of data.
* this type of analytical activity is important for overall quality & health of an operational data store (ODS) or DWH.
* rather than use a set of complicated queries or rely on third-party product, v have a data profiling task.
* it is more of an ad hoc tool for placement in design-time package that will be run manually outside of the scheduled process
* task doesn't have built in conditional workflow logic (technically u can use XPath queries on the results)

following profiling operations can be done using this task: [sanb1]
1. candidate key profile request
2. column length distribution profile request
3. column null ratio profile request
4. column pattern profile request
5. column statistics profile request
6. column value distribution profile request
7. functional dependency profile request
8. value inclusion profile request



Execute Package
* let u execute a different int p/c [sanb1]
* p/c containing 'Execute Package' task is parent p/c & p/c being executed is child p/c [sanb1]
* enables u to build SSIS solutions called parent packages that execute other packages called child packages.
* separating packages into discrete functional workflows enables shorter development & testing cycles and facilitates best development practices.


Execute Process
* enable us to execute a program or a batch file as part of int services p/c. [sanb1]
* will execute a windows application or console application inside the control flow.


Execute SQL
* enables us to execute SQL statements or stored procedures. [sanb1]
* one of the most widely used tasks in SSIS for interacting with RDBMS data source.
* variables can be used for input, output, or input/output parameters & return values. [sanb1]
* used for all sort of things

File System
* is a configurable GUI component that performs file operations available in System.IO.File .Net class.
* If u are used to coding in VBScript, this is an out-of-the-box replacement for the VBScript utility classes that you used to write using the COM-based FileSystemObject.
* In either case, the File System Task can perform basic file operations such as the following:
  1. copy d/y
  2. copy file
  3. create d/y
  4. delete d/y
  5. delete d/y content
  6. delete file
  7. mv d/y
  8. mv file
  9. rename file
  10. set attributes
