Aggregate t/n
* enable us to combine information from multiple records in data flow into a single value. [san1]
* functions same way as aggregation functions & GROUP BY in sql [san1]
* following aggregations r available: [san1]
  1. Average
  2. Count
  3. Count Distinct
  4. Maximum
  5. Minimum
  6. Sum

Cache t/n
* enables us to populate a cache that will be subsequently used by a lookup t/n [san1]


Lookup t/n
* works similarly to 'Fuzzy' lookup t/n [san1]
    * difference is lookup t/n requires exact matches, rather than using similarity scores.
* selection of source lookup table and column mapping is done in much same way as for Fuzzy lookup t/n.


multicast t/n
* let us take a single data flow & use it as input to several data flow t/ns or data flow destination items. [sanb2]
* simply copies the data flow & sends it to multiple directions. [sanb2]

percentage sampling t/n
* enables us to split the data flow into two seperate data flows based on %. [sanb2]
* useful when we want to create a small sample of a larger set of data for testing or for training a data mining models. [sanb2]
* selects rows at random from input data flow. [sanb2]
    * this random sampling provides a more representative sample of entire data flow than simply selecting TOP N rows. [sanb2]


row sampling t/n
* enables us to split the data flow into two seperate data flows based on no. of rows. [sanb2]
* works same as percentage sampling t/n
    * only difference is it determines no. of rows in the sample based on requested row count, rather than a requested %. [sanb2]



union all
* let us merge several data flow into single data flow. [sanb2]
