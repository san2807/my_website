JOINS

* JOIN table operator operates on two input tables.
* 3 fundamental types of JOINS :  Cross Joins
                                                        Inner Joins
                                                        Outer Joins

CROSS Joins
* simplest types of join.
* implements only one logical query processing phase - a Cartesian product of the two
        * i.e. each row from one input is matched with all rows from other.
        * if u have m rows in one table & n rows in another, u get m*n rows in the result.


INNER Joins
* applies two logical query processing phases - applies Cartesian product between two input tables
                                                                            and then filters rows based on predicate you specify.



OUTER Joins
