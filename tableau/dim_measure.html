Dimension (d/n)

* d/n r qualitative data, such as name or date
* by default, t/u automatically classifies data that contains qualitative or categorical information as d/n
        *for e.g. any field with text or date value
* these fields generally appear as column headers for rows of data, such as customer name or order date
* t/u does not agreegate d/ns,   [1]
    * if u want field value to be aggregated that must be measure [1]

Measure (m/e)
* m/e r quantitative numerical data.
* m/e r values tht r aggregated i.e. they can be summed, averaged, counted or can have minimum or maximum values. [sanb1] * by default, t/u treats any field containing this kind of data as measure (sales or transaction profit)
* data that is classified as measure can be aggregated based on dimension

Continuous (green) (m/e)
* continuous field can contain infinite no. of values
* colored GREEN

Discrete (blue) (d/n)
* fields contain a finite no. of values
