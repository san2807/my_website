*join & blend r two different ways to link related data.
* always start by adding primary t/b

Join
* performed to link tables within single data source
* done at row level
* involves single query to a single data source
* when u connect to d/b in t/u, u see all tables, always start by adding primary t/b [sanb1]
* if refrential integrity has been defined in d/b, t/u will automatically create joins as u add additional t/bs. [sanb1]
    * else it will try to join on column name basis. [sanb1]
    * in any case, u may adjust joins as needed. [sanb1]

Blend (b/l)
* performed to link together multiple data sources.
* data sources can be of different types
    * for e.g. blend data from Oracle with data from Excel
                    blend data from Google Analytics with MS Access
* done at aggregate level [sanb1]
* involves different queries sent to each data source. [sanb1]

++++++++++++++++++++++++++++++++
* b/ling allows u to use data from multiple data sources in same view. [sanb1]
* these sources may b of different types. [sanb1]
*b/ling allows to compare data at different level of details (LOD) [sanb1]


difference b/w joining & b/ling
* joins r accomplished by single query & results r matched row by row.
*b/ling occurs by issuing two or more seperate queries & then blending together aggregate results.



Filter
* often, u will want to filter data in t/u in order to
        * perform analysis on subset of data,
        * narrow ur focus
        * drill into detail

types of filter in t/u
  1. Data Source filter
  2. Extract Filter
  3. Custom SQL Queries Filter
