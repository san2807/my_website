Role-Based s/y
* user have roles through which they gain access to functions and data
* HCM defined 5 types of roles:
        1. Data roles
        2. Abstract roles
        3. Job roles
        4. Aggregate rol
        5. Duty roles

Data Roles
    * u define the data scope of a data role in one or more security profiles
    * u define all the data role locally n assign them directly to users


Abstract Roles
    * represent the worker's role in the enterprise independently of the job you hire the worker to do.
    * 3 main predefined:
            1. Employee
            2. Contingent Worker
            3. Line Manager
    * can also create abstract role

Job Roles
    * represent the job that a worker has to perform
    * typically u include job roles in data roles & assign those data roles to user

Aggregate Privileges
    * combine functional privilege for an individual task or duty with relevant data s/y policies
    * they are predefined & u can't edit them
    * u don't assign it directly to user

Duty Roles
    * represents logical grouping of privileges
    * u don't assign it directly to user
