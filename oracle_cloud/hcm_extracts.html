HCM Extracts

Extract Definition
    * refers to complete setup of an e/x

Data Groups (d/g)
    * represent a business area or logical entity, for e.g. person, assignment or benefits
    * define one data group as primary / root data group (which is starting point of the extraction)
    * d/g can be filtered using d/g criteria
    * need UE for creating d/g
   

Data Group Connections
    * capture association b/w current d/g and parent d/g
    * form hierarchical relationships among the d/gs

Extract Records
    * represent grouping of related data or a physical collections of all fields required in e/x

Attributes
    * individual fields inside the e/x record

Extract Types
    * determines the purpose of e/x
    * also determines the parameters that r automatically generated.
    * e/x types:
            1. Full profile
            2. Payroll interface
            3. Payments
            4. Benefit Carrier
            5. Archive Retrival
            6. EOY Archive
            7. HR Archive
            8. Payroll Archive
            9. Other Payroll Archive



Flow Pattern
* exists for each extract or report process.
* defines submission parameters, flow tasks, and task sequence.
*

Flow Tasks

Flow Pattern Parameters

Parameter Basis
    * controls how application derives the value for the p/m
    * types of p/m basis
        1. Use specified value
        2. Bind to Context
        3. Bind to Flow Parameter
        4. Bind to Flow Task p/m
        5. Bind to Task p/m
        6. No value specified
        7. Post SQL Bind
        8. SQL Bind
