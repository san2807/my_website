f/f functions

d/b items
    * exists in the application d/b & have computer code associated with them.
    * d/b items r read-only variables
    * can't change d/b item value within formula
    * can use it in f/f & HCM extracts
    * types
            * static
            * dynamic
            * array


contexts:
* context is info that is always available to each formula type at run
* contexts are available for certain formula types
* e.g. of contexts:
    * EFFECTIVE_DATE for effective date the formula is executing
    * PAYROLL_ID for running payroll
    * PERSON_ID identifying the persorm for whom the formula is exectuing
* context values act as SQL bind values when d/b item values are fetched from the d/b
* they can also be passed into formula fuction calls
* methods for context are:
    * GET_CONTEXT:             to find the value from the context
    * CONTEXT_IS_SET:         to find whether a context is set
    * SET_CONTEXT:              to set one or more context

array processing:

looping:
* WHILE loop type is supported

working staging area:
* the wsa is a mechanism for storing global values across formulas
* the values are accessed by name
* names are case-independent
* there r 4 wsa call methods:
    * WSA_EXISTS
    * WSA_DELETE
    * WSA_SET
    * WSA_GET

f/f functions:
* text functions
*numeric fuctions
* date functions

calling f/f from a f/f:
* f/f can be called from another f/f
* there r 4 methods available:
    * IS_EXECUTE      (f/f name)
    * EXECUTE           (f/f name)
    * SET_INPUT        (input[,value])
    * GET_OUTPUT    (output, default-value)
