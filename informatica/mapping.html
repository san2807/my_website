Mapping

* is a set of source definition, target definition linked by t/n objects that define rules for data t/n.
* represent data flow b/w sources & targets.
* when int services runs a s/n, it uses instructions configured in mapping to read, transform & write data.
* every mapping must contain:
  1. source definition
  2. t/n
  3. target definition
  4. links

* mapping can also contain one or more mapplets.
* mapping desinger displays objects in 3 different views:

  1. iconized
  2. normal
  3. edit



Mapplet
* is a reusable object
* can create in Mapplet Designer
* contains set of t/ns & lets u reuse t/n logic in multiple mappings.
* when u use a mapplet in a mapping, u use an instance of mapplet.


Mapping Parameter (p/m)
* represent constant value that u can define before running a s/n.
* retain same value throughout the s/n
* declare & use p/m in mapping/mapplet, then define value of p/m in p/m file


Mapping Variable (v/a)
* unlike p/m, mapping v/a represents a value that can change through s/n.
* int service saves value of v/a to rep at end each successful s/n run & uses tht value next time u run s/n
* declare & use v/a in mapping/mapplet, then use v/a function in mapping to change value
* at beginning, int service evaluates references to v/a to determine start value.
* at end of successful s/n, int service saves final value of v/a to the rep
* next time u run s/n int service evaluates references to v/a to saved value
* to override saved value:
        * define start value of v/a in a p/m file
                            or
        * assign value in pre-s/n v/a assignment in s/n properties.
