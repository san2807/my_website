Router Transformation

* similar to filter transformation as both allow u to use a condition to test data.
* filter test data for one condition and drops rows of data that do not meet condition.
* router test data for one or more conditions and give you option to route rows of data that do not meet any of conditions to a default o/p group.
* to test same input data based on multiple conditions, use a ROUTER in mapping instead of creating multiple filter transformations to perform same task.

* when u use ROUTER in a mapping, int service processes incoming data only once, but when u use FILTER in a mapping, int service processes incoming data for each transformation.

ROUTER consists of :
* Input Ports
* Input Group
* Output Ports
* Output Groups
        * User-Defined output Groups
        * Default output Groups

user-defined output groups
* created by user to test a condition based on incoming data.
* consists of output ports & a group filter condition.
* create on user-defined group for each condition that you want to specify.
* int service evaluates each incoming row with condition.
* int service test conditions of each user-defined groups before sending the row to default group.
* order of evaluation depends on order of connect output groups.
* int service processes user-defined groups that are connected to a transformation or target in mapping.
* int service processes user-defined groups that are not connected only if default group is connected to a transformation or target in mapping.
* if row satisfies condition of multiple groups, int service passes this row multiple times.
