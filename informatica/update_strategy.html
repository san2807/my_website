Update Strategy Transformation

* when designing a DWH, u need to decide what type of information to store in targets.
* as part of design, u need to determine whether to maintain all historic data or just most recent changes.


* model u choose determines how u handle changes in existing rows.
* in powercenter, u set update strategy at two levels:
    * within a session
    * within a mapping
* can also use CUSTOM transformation to flag rows for insert, delete, update or reject.

setting update strategy
* in mapping : to control how rows are flagged for insert, update, delete or reject within a mapping, add UPDATE STRATEGY transformation.
        * essential if u want to flag rows destined for same target for different database operations, or if u want to reject rows.
* in session : can flag all rows for insert, delete or update
                      can select data driven option (where int service follows instructions coded into update strategy transformations within session mapping).

flagging rows within a mapping
* for greatest degree of control over update strategy, add it to a mapping.
* most important feature is its update strategy expression, used to flag individual rows for insert, delete, update or reject.
* constant for each database operation and their numeric equivalent:
        Insert            DD_INSERT        0
        Update          DD_UPDATE    1
        Delete           DD_DELETE     2
        Reject           DD_REJECT      3

* any other value will be treated as INSERT
