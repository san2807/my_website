Expression t/n

* use to calculate values in a single row.
* for e.g. u might need to adjust salaries, concatenate first and last names or convert strings to numbers.
* can also use to test conditional statements before u pass results to a target or other transformations.
* use to perform non-aggregate calculations. (to perform calc on multiple rows, such as sums or averages, use AGGREGATOR t/n)

calculate values

        * input or input/output ports
        * output ports


Filter t/n
* use to filter out rows in a mapping
* as an active t/n, it may change the no. of rows passed through it.
* it drops rows that do not meet conditions.
* can filter data based on one or more conditions.
* it returns TRUE or FALSE for each row that int service evaluates.
    * for each row that returns TRUE, int service pass through t/n
    * for each row that returns FALSE, int service drops and writes a message to session log.
* can not concatenate ports from more than one t/n into filter t/n
* place filter t/n as close to sources in mapping as possible to maximize session performance.

* there is an alternative to filter t/n. that is source qualifier.  [sanref1]
    * source qualifier can also filter the records but there is constraint that it only works for relational sources.

* see some scenarios given in [sanref1]
