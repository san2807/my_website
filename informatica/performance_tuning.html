Performance Tuning

* first step is to identify performance bottlenecks.
* it can occur in source d/bs, target d/bs, mapping, session & system
* strategy should be
        identify a performance bottleneck,
        eliminate it
        & then identify next performance bottleneck until u r satisfied with performance.
* look in below order
  1. Target
  2. Source
  3. Mapping
  4. Session
  5. System
Thread Statistics

Optimizing Target
  1. flat file target
  2. dropping index & key constraints
  3. increasing d/b checkpoints intervals
  4. using bulk loads
  5. using external loaders
  6. minimizing deadlocks
  7. increasing d/b n/w packet size
  8. optimizing ORACLE d/b target
Optimizing Source
  1. optimizing query
  2. using conditional filters
  3. increasing d/b n/w packet size
  4. connecting ORACLE d/b source
  5. using TERADATA fast export
  6. using tempdb to join in SYBASE or MS SQL Server
