Union Transformation

* multiple input group transformation use to merge data from multiple pipelines or pipeline branches into one pipeline branch.
* merges the data from multiple sources similar to UNION ALL SQL statement to combine results from two or more SQL statements.
* Similar to UNION ALL statement, UNION transformation does not remove duplicate rows. to remove duplicate rows u must add another transformation like ROUTER or FILTER.
* int service processes all input groups in parallel.
* it concurrently read sources connected to UNION transformation and pushes blocks of data into input groups of transformation.
* it processes blocks of data based on order it receives blocks from int services.
* developed using CUSTOM transformation

Rules
* can create multiple input groups but only one output group.
* all input groups and output group must have matching ports. precision, datatype and scale must be identical across all groups.
* cannot use a SEQUENCE generator or UPDATE STRATEGY upstream from a union transformation.
* union transformation doesn't generate transactions.
