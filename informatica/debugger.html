Debugger

* can debug a valid mapping to gain troubleshooting information about data and error conditions.
* to debug a mapping, configure and run debugger from within the mapping designer.
* debugger uses a session to run mapping on the int service.
* when u run debugger, it pauses at breakpoints and you can view and edit transformation output data.

when u can run debugger:
* before u run a session
* after u run a session

debugger session types:
* use an existing non-reusable session.
* ===========reusable=========
* create a debug session instance.

debug process
* create breakpoints
* configure debugger
* run debugger
* monitor debugger
* modify data and breakpoints
