dimension table structure
* every d/n table has a single primary key column. [sanb1]
* this primary key is embedded as foreign key in associated fact table
    where d/n row's descriptive context is exactly correct for that fact table row.[sanb1]

slowing changing dimension (scd)
  1. type 0 : Retain original
  2. type 1 : Overwrite
  3. type 2 : add new row
  4. type 3 : add new attribute
  5. type 4 : add mini-dimension
  6. type 5 : add mini-dimension & type 1 outtrigger
  7. type 6 : add type 1 attributes to type 2 dimension
  8. type 7 : dual type 1 and type 2 dimensions
