Oracle Digital Assistant


Skills
* indiviual chatbots that are designed to interact with users and fulfill specific types of tasks

Intents
* as specific use case that the bot is able to understand and implement (e.g. display the payslip). There may be may different ways the user could express that intent.
* categories of actions or task user expect your skill to perform for them.

utterrance
A way of expressing an intent.
for e.g. "I want to view my payslip" and "How much was I paid" may both express the same intent to the digital assistant.

Entities
* variables that identify key pieces of information  from user input that enable the skill to fulfill a task

Components
* provide ur skill with various f/ns so that it can respond to users.

Dialog flow
* OBotML is Digital Assistant's own implementation of YAML.

Channel
*one of various mechanism whtour which a user can interact with digital assitant,

Skill Store

Security




Visual Dialog Mode: [Link]
VDM is the new standard in ODA for building dialog flows

A skill have multiple flows defined
variables have a flow scope. as you enter the flow variable is set and you exit variable is reset
parameters new feature that define the contract b/w a flow and its outer surrounding
events are like default transitions in YAML dialog mod and are mapped to flows
states still are the conversation script followed in a bot - user interaction

* if you start building skill using visual dialog mode, you can't access YAML
